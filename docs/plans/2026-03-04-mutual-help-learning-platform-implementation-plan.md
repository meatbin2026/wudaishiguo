# 学习资料互助平台 MVP 实施计划

- 日期：2026-03-04
- 关联设计：`docs/plans/2026-03-04-mutual-help-learning-platform-design.md`
- 目标：4 周内上线可用 MVP

## 1. 里程碑

1. M1（第 1 周结束）：可发布任务并冻结积分，可提交答案。
2. M2（第 2 周结束）：可选最佳并完成积分结算，支持 72 小时自动结算。
3. M3（第 3 周结束）：可举报、可审核、具备基础风控与信誉分。
4. M4（第 4 周结束，可选）：优化性能与体验，准备灰度上线。

## 2. 工作分解（按模块）

### 2.1 账号与权限
- 任务：
1. 用户注册/登录（JWT + refresh token）
2. 基础用户资料接口
3. 权限中间件（普通用户/管理员）
- 验收标准：
1. 登录态可在前后端正常续期
2. 受限接口可正确拦截未授权访问

### 2.2 任务系统
- 任务：
1. 发布任务 API（校验字段+余额）
2. 任务列表与详情 API（分页、筛选、排序）
3. 任务状态机（open/closed/auto_settled/cancelled）
- 验收标准：
1. 发布时冻结积分并写流水
2. 状态流转不可越级或重复提交

### 2.3 回答系统
- 任务：
1. 多人提交答案 API
2. 答案编辑/删除规则（仅提交后短时间可改，避免争议）
3. 答案排序（先按质量分，再按时间）
- 验收标准：
1. 单任务可容纳多条合规答案
2. 任务关闭后不可新增答案

### 2.4 结算系统
- 任务：
1. 选最佳答案接口
2. 结算事务（任务状态、钱包、流水原子提交）
3. 自动结算 worker（72 小时）
- 验收标准：
1. 任意任务只可结算一次
2. 并发下余额与流水一致

### 2.5 积分中心
- 任务：
1. 钱包总览 API（可用/冻结）
2. 积分流水查询 API
3. 流水类型规范（freeze/reward/refund/penalty）
- 验收标准：
1. 每次积分变更都可追踪到业务单据

### 2.6 风控与举报
- 任务：
1. 发布与回答关键词过滤
2. 域名黑白名单校验
3. 举报提单与后台处理流
- 验收标准：
1. 高风险内容能拦截或进入审核
2. 处理动作可审计（audit logs）

### 2.7 前端页面
- 任务：
1. 首页/任务广场
2. 发布页/详情页/我的任务
3. 积分中心/举报页
4. 管理后台（最小版）
- 验收标准：
1. 完整走通发布->回答->选最佳->结算链路
2. 关键页在移动端可用

## 3. 数据库与迁移计划

1. 建表顺序：`users -> tasks -> answers -> point_transactions -> point_ledger -> reports -> audit_logs`
2. 迁移策略：
- 初版采用 SQL migration 脚本管理
- 每次 schema 改动附回滚脚本
3. 一致性策略：
- 结算相关接口全部使用事务
- 为 `task_id` + `status=paid` 添加唯一约束，防重复结算

## 4. API 清单（MVP）

1. `POST /auth/register`
2. `POST /auth/login`
3. `GET /tasks`
4. `POST /tasks`
5. `GET /tasks/:id`
6. `POST /tasks/:id/answers`
7. `POST /tasks/:id/select-best`
8. `GET /me/tasks/published`
9. `GET /me/tasks/answered`
10. `GET /me/points`
11. `GET /me/points/ledger`
12. `POST /reports`
13. `GET /admin/reports`
14. `POST /admin/reports/:id/resolve`

## 5. 测试计划

1. 单元测试：
- 钱包冻结与结算
- 信誉分计算
- 状态机合法性校验
2. 集成测试：
- 端到端业务主链路
- 并发结算冲突
- 自动结算任务
3. 回归检查：
- 权限绕过
- 非法内容拦截
- 举报与处罚一致性

## 6. 风险与应对

1. 风险：内容合规争议
- 应对：前置提示 + 举报通道 + 管理员复核
2. 风险：并发造成重复结算
- 应对：数据库事务 + 唯一约束 + 幂等键
3. 风险：冷启动内容不足
- 应对：先引入种子任务与任务模板

## 7. Definition of Done

1. 用户可在不上传文件前提下完成一次完整悬赏求助。
2. 平台能在 72 小时无人处理时自动结算。
3. 积分账本与交易单据可核对一致。
4. 举报流程可闭环，违规内容可处理。
